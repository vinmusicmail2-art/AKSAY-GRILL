## [Апрель 2026] — Мероприятия в зале и кейтеринг

### Добавлено: страница «Мероприятия в зале» (`/events`)

**Файлы:** `models.py`, `forms.py`, `mailer.py`, `app.py`,
`templates/events.html`, `templates/admin/events.html`,
`templates/admin/dashboard.html`, `templates/index.html`, `assets/events.png`

- Новый публичный роут `GET /events` и `POST /events` — страница бронирования
  зала ресторана для торжеств (юбилей, ДР, свадьба, корпоратив, детский
  праздник, поминальный обед, другое).
- Каталог `EVENT_TYPES` (7 типов) в `models.py`.
- Модель `HallReservation` (таблица `hall_reservations`, 18 полей):
  контактное лицо, компания, телефон, e-mail, тип мероприятия, число гостей
  (2–300), дата, время начала, длительность (часов, опц.),
  флаги «нужно оформление зала» и «нужна помощь с меню», комментарий, IP,
  статус обработки и аудитные поля.
- Форма `HallReservationForm` с валидацией (WTForms + CSRF).
- Герой с изображением банкетного зала (`assets/events.png`, AI-генерация,
  16:9), тёплый терракотовый оверлей.
- Три плитки преимуществ: «До 300 гостей», «Меню под событие»,
  «Оформление».
- Карточки типов мероприятий с кнопками «Забронировать» (автовыбор
  типа в форме через `onclick`).
- Форма бронирования с чекбоксами «Оформление зала» и «Помощь с меню»
  в виде кликабельных строк (check-row).
- Страница согласия на обработку ПД.
- Кнопка «Обсудить событие» в карточке «Мероприятия» на главной теперь
  ведёт на `/events` (была `<button>`, стала `<a>`).
- Админ-роут `GET /admin/events` — список заявок с фильтром
  «В работе / Обработанные / Все»; бейджи «Оформление зала» / «Помощь с меню».
- Админ-роут `POST /admin/events/<id>/toggle` — тумблер обработки заявки
  (фиксирует `processed_at` и `processed_by`).
- Плитка «Заявки на банкеты» на дашборде с бейджем `pending_events`.
- Функции `_format_hall_email`, `send_hall_notification`,
  `send_hall_notification_async` в `mailer.py` — письмо администратору
  при новой заявке (SMTP, фоновый поток).

## [Апрель 2026] — E-mail уведомления и бизнес-ланчи

### Добавлено: e-mail уведомления (`mailer.py`)

**Файлы:** `mailer.py`, `models.py` (SiteText-ключи), `app.py`,
`templates/admin/email_settings.html`, `templates/admin/dashboard.html`

- Новый модуль `mailer.py` — отправка e-mail через стандартную библиотеку
  `smtplib`; поддержка порта 465 (SSL) и 587 (STARTTLS).
- Все уведомления отправляются в фоновом потоке (`threading.Thread`) —
  не блокируют HTTP-ответ.
- Безопасный fallback: если SMTP не настроен или отключён, заявка
  сохраняется в БД, ошибка только логируется.
- Два управляющих ключа в `site_texts`: `notify_email_recipient` (адрес
  получателя) и `notify_email_enabled` (вкл/выкл).
- Функции: `send_order_notification` / `send_order_notification_async`
  (бизнес-ланчи), `send_test_email`, `smtp_status`.
- Подробные HTML-письма в фирменном стиле (терракотовый header, таблица
  полей, кнопка «Открыть в админке»).
- Страница `/admin/email-settings`:
  - статус SMTP (настроен / не настроен, без раскрытия пароля),
  - поле адреса получателя,
  - переключатель «Уведомления включены»,
  - поле и кнопка отправки тестового письма.
- Плитка «Уведомления на e-mail» на дашборде.
- SMTP-параметры: переменные окружения `SMTP_HOST`, `SMTP_PORT`,
  `SMTP_USER`, `SMTP_PASSWORD`, `SMTP_FROM` (не заполнены до настройки).

## [Ранее] — Базовая инфраструктура

### Добавлено: перевод на Flask + SQLite

- Статический HTML преобразован в Flask-приложение (`app.py`, `db.py`,
  `models.py`).
- SQLite-база `data.db` (создаётся автоматически при первом запуске).
- Gunicorn в качестве сервера (workflow `Start application`).
- Tailwind CSS через CDN, Google Fonts (Montserrat), Material Symbols.
- Логотип `assets/logo.jpg`, фон `assets/wood.png`, герой `assets/hero-grill.png`.
- Политика конфиденциальности (`/privacy.html`).
- Яндекс.Карта в карточке «Как нас найти».

### Добавлено: авторизация и журнал входов

- Модель `Admin` с bcrypt-хешем пароля и поддержкой Flask-Login.
- Модель `LoginLog` — журнал всех попыток входа (IP, User-Agent,
  успех/отказ) для соответствия 152-ФЗ.
- `/admin/setup` — одноразовое создание первого администратора (404 после).
- `/admin/login`, `/admin/logout` — вход/выход с CSRF.
- Журнал входов на дашборде (последние 50 записей).

### Добавлено: редактор текстов сайта

- Модель `SiteText` — хранилище текстовых блоков сайта.
- Каталог `SITE_TEXT_CATALOG` в `models.py` (key, label, kind, default).
- Context-processor `inject_site_texts` — словарь `texts` доступен
  во всех шаблонах автоматически.
- `/admin/texts` — страница редактирования всех текстовых блоков
  главной страницы (заголовки, теги, контакты, карта, копирайт и др.).
- Начальное сидирование дефолтных значений при первом запуске.
