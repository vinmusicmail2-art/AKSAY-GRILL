## 1. Вход в административную панель

**Адрес:** `ваш-домен.ru/admin`

При первом переходе в `/admin` система перенаправит вас на страницу входа `/admin/login`.

**Поля для входа:**
- **Логин** — имя пользователя, заданное при первоначальной настройке
- **Пароль** — пароль администратора

> **Первый запуск:** Если вы открываете сайт впервые и администратор ещё не создан, система откроет страницу `/admin/setup`. Введите логин и пароль для создания первого администратора. После создания страница setup **больше не будет доступна**.

После входа вы попадёте на дашборд. Для выхода — кнопка **«Выйти»** в верхнем меню.

## 2. Главная страница (Дашборд)

**Адрес:** `/admin`

На дашборде отображается:

| Блок | Описание |
|------|----------|
| **Новые заказы доставки** | Количество необработанных заказов из корзины |
| **Новые заявки бизнес-ланч** | Количество необработанных запросов на бизнес-ланч |
| **Новые заявки кейтеринг** | Количество необработанных кейтеринговых заявок |
| **Новые заявки мероприятия** | Количество необработанных заявок на аренду зала |
| **История входов** | Последние 20 входов с указанием даты, времени и IP-адреса |

Блоки со счётчиками являются ссылками — нажмите на любой из них, чтобы перейти к соответствующему разделу заказов.

## 3. Заказы доставки

**Адрес:** `/admin/delivery-orders`

Сюда поступают заказы, оформленные через корзину на главной странице сайта (кнопка **«Заказать доставку»**).

### Что содержит каждый заказ

| Поле | Описание |
|------|----------|
| **Имя** | Имя клиента |
| **Телефон** | Контактный номер |
| **E-mail** | Электронная почта (если указана) |
| **Адрес доставки** | Куда доставлять |
| **Состав заказа** | Список блюд с количеством и ценами |
| **Итоговая сумма** | Общая стоимость заказа |
| **Комментарий** | Пожелания клиента |
| **Дата и время** | Когда поступил заказ |
| **IP-адрес** | Технический идентификатор клиента |

### Как обработать заказ

1. Позвоните клиенту по указанному телефону для подтверждения
2. Уточните время доставки
3. Нажмите кнопку **«Отметить обработанным»** — заказ переместится в раздел «Выполненные»
4. При необходимости нажмите снова — статус вернётся обратно к «Новым»

> **Важно:** Система только принимает заказы. Расчёт доставки и работа с курьером происходит вне сайта — по телефону.

## 4. Бизнес-ланчи

**Адрес:** `/admin/business-lunches`

Сюда поступают заявки на корпоративные бизнес-ланчи, оформленные через страницу `/business-lunch`.

### Что содержит заявка

| Поле | Описание |
|------|----------|
| **Имя** | Контактное лицо |
| **Компания** | Название организации |
| **Телефон / E-mail** | Контакты |
| **Количество персон** | Сколько человек |
| **Дата доставки** | На какую дату заказ |
| **Время доставки** | Желаемое время |
| **Адрес доставки** | Куда привезти |
| **Выбранные комбо** | Набор(ы) блюд: Лёгкий, Сытный, Гриль, Вегетарианский |
| **Комментарий** | Дополнительные пожелания |

### Как обработать

1. Свяжитесь с клиентом для уточнения деталей
2. Согласуйте состав, количество порций и точное время доставки
3. Нажмите **«Отметить обработанным»** после звонка/договорённости

## 5. Кейтеринг

**Адрес:** `/admin/catering`

Сюда поступают заявки на выездное обслуживание мероприятий — страница `/catering`.

### Что содержит заявка

| Поле | Описание |
|------|----------|
| **Имя / Компания** | Контактные данные организатора |
| **Телефон / E-mail** | Для обратной связи |
| **Формат мероприятия** | Банкет, Фуршет, Выездной гриль, Кофе-брейк, Поминки, Свадьба, Другое |
| **Количество гостей** | Ожидаемое число участников |
| **Дата и время** | Когда планируется мероприятие |
| **Место проведения** | Адрес или описание площадки |
| **Бюджет на гостя** | Ориентировочный бюджет (руб./чел.) |
| **Комментарий** | Дополнительные пожелания |

### Как обработать

1. Свяжитесь с клиентом — кейтеринг требует предварительного согласования меню, персонала и логистики
2. После переговоров нажмите **«Отметить обработанным»**

**Адрес:** `/admin/events`

Сюда поступают заявки на проведение мероприятий в зале ресторана — страница `/events`.

### Что содержит заявка

| Поле | Описание |
|------|----------|
| **Имя / Компания** | Организатор мероприятия |
| **Телефон / E-mail** | Контакты |
| **Тип мероприятия** | Юбилей, День рождения, Свадьба, Корпоратив, Детский праздник, Поминки, Другое |
| **Количество гостей** | Ожидаемое число участников |
| **Дата и время** | Дата начала |
| **Продолжительность** | На сколько часов нужен зал |
| **Нужен декор** | Да / Нет |
| **Помощь с меню** | Нужна ли помощь в составлении меню |
| **Комментарий** | Дополнительные пожелания |

### Как обработать

1. Уточните доступность зала на указанную дату
2. Свяжитесь с клиентом и согласуйте детали
3. Нажмите **«Отметить обработанным»**

## 7. Быстрые заявки

Быстрые заявки поступают с кнопки **«Оставить заявку»** в секции доставки на главной странице. Это упрощённая форма — клиент оставляет только имя, телефон и адрес. Обработка аналогична остальным разделам.

## 8. Редактирование текстов сайта

**Адрес:** `/admin/texts`

Здесь можно изменить тексты, отображаемые на сайте, **без привлечения разработчика**.

### Доступные для редактирования тексты

#### Главная страница
- Название сайта, слоган, текст в герое (заголовок, описание мясных и овощных блюд)
- Текст кнопок CTA («Заказать доставку», «Посмотреть отзывы»)
- Текст в подвале страницы (copyright)

#### Контакты в шапке
- Адрес ресторана
- Номер телефона
- Часы работы

#### Реквизиты оператора (для политики конфиденциальности)
- Название организации, ИНН, ОГРНИП, дата регистрации
- Юридический адрес, налоговый орган, e-mail, телефон

#### Страницы О нас / Кейтеринг / Мероприятия / Бизнес-ланч
- Подзаголовки и описания на каждой странице

#### Карта
- Адрес, координаты (широта/долгота), масштаб карты

### Как редактировать

1. Перейдите в `/admin/texts`
2. Найдите нужное поле
3. Отредактируйте текст
4. Нажмите **«Сохранить»**

Изменения применяются на сайте **немедленно**.

## 9. Настройка e-mail уведомлений

**Адрес:** `/admin/email-settings`

При каждом новом заказе или заявке система может автоматически отправлять уведомление на указанный e-mail.

### Настройки

| Параметр | Описание |
|----------|----------|
| **E-mail для уведомлений** | Адрес, на который будут приходить оповещения о новых заявках |
| **Уведомления включены** | Главный переключатель: Да / Нет |

### Тестирование

На странице email-настроек есть кнопка **«Отправить тестовое письмо»**. Используйте её, чтобы убедиться, что письма доходят.

### SMTP-сервер (для технического специалиста)

Настройки SMTP задаются через переменные окружения на сервере (не через интерфейс):

| Переменная | Значение |
|------------|----------|
| `SMTP_HOST` | Адрес почтового сервера (например, `smtp.yandex.ru`) |
| `SMTP_PORT` | Порт: `465` (SSL) или `587` (STARTTLS) |
| `SMTP_USER` | Логин почтового ящика |
| `SMTP_PASSWORD` | Пароль от почтового ящика |
| `SMTP_FROM` | E-mail отправителя (обычно = SMTP_USER) |

> Если SMTP не настроен — сайт работает в штатном режиме, уведомления просто не отправляются. Ошибок для клиентов не возникает.

## 10. Безопасность и рекомендации

### Пароль администратора
- Используйте надёжный пароль (минимум 12 символов, буквы + цифры + спецсимволы)
- Не сообщайте пароль третьим лицам
- При подозрении на компрометацию — смените пароль через `/admin/setup` (доступно только при удалении всех записей администраторов из базы данных, иначе — через разработчика)

### Работа с заказами
- Обрабатывайте заказы как можно быстрее — клиенты ожидают обратной связи в течение **15–30 минут**
- Используйте кнопку «Обработан» только **после** того, как связались с клиентом
- Обработанные заявки хранятся в системе и доступны для просмотра

### Сессия
- Браузер автоматически разлогинивает вас при закрытии вкладки (если не выбрано «Запомнить»)
- Всегда нажимайте **«Выйти»** при использовании общего компьютера

### Резервные копии
- База данных хранится в файле `data.db` в папке сайта
- Рекомендуется регулярно делать резервную копию этого файла

## 11. Что делать, если что-то пошло не так

### Не приходят e-mail уведомления
1. Перейдите в `/admin/email-settings`
2. Убедитесь, что уведомления **включены** и указан правильный e-mail
3. Нажмите **«Отправить тестовое письмо»**
4. Если письмо не пришло — проверьте папку «Спам»
5. Если проблема сохраняется — обратитесь к разработчику для проверки SMTP-настроек

### Клиент говорит, что не смог отправить форму
1. Попросите клиента обновить страницу и попробовать снова
2. Уточните, какое сообщение об ошибке отображается
3. Примите заказ по телефону вручную

### Счётчик на дашборде не уменьшается
- Убедитесь, что вы нажали кнопку **«Отметить обработанным»** (а не просто просмотрели заявку)
- Обновите страницу браузера

### Сайт не открывается
- Обратитесь к хостинг-провайдеру или разработчику
